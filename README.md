# For Future

这是一个两人共同维护的每日打卡仓库。我们把每天写过的代码、刷过的题、学到的知识和阶段性复盘都留在这里，让长期积累变得可见。

## 仓库目标

- 保持每日学习和练习的节奏
- 记录算法、项目、课程、阅读等学习成果
- 互相监督，互相鼓励，降低半途而废的概率
- 让未来回头看时，能清楚看到自己一步一步走过来的轨迹

## 目录建议

建议每个人使用一个独立目录，并按日期继续拆分：

```text
for_future/
├── README.md
├── main.py
├── hyt/
│   └── 0428/
│       ├── leetcode_124.py
│       ├── leetcode_173.py
│       └── leetcode_222.py
└── your_name/
    └── 0428/
        └── note.md
```

推荐命名方式：

- 个人目录：使用姓名缩写、昵称或 GitHub 用户名
- 日期目录：使用 `MMDD`，例如 `0428`
- 题目文件：使用 `leetcode_题号.py`，例如 `leetcode_124.py`
- 笔记文件：使用 `note.md`、`summary.md` 或具体主题名

## 每日打卡内容

每天可以提交以下任意一种或多种内容：

- 算法题代码
- 学习笔记
- 项目代码
- Bug 记录和解决过程
- 阅读总结
- 当日复盘

打卡不要求每天都很完美，但尽量做到真实、连续、可回顾。

## 打卡模板

可以在每天的目录里创建一个 `note.md`：

```markdown
# 0428 打卡

## 今日完成

- LeetCode 124
- LeetCode 173
- LeetCode 222

## 今日收获

- 复习了二叉树递归
- 熟悉了中序遍历迭代写法

## 明日计划

- 继续刷二叉树相关题目
- 整理今天代码中的边界条件
```

## 提交规范

建议使用清晰的 commit message，方便之后回顾：

```text
daily: hyt 0428 leetcode
daily: add 0428 notes
fix: update leetcode 124 solution
docs: update README
```

## 协作约定

- 每个人维护自己的目录，避免误改对方文件
- 如果需要调整公共文件，先在提交信息里写清楚原因
- 每天尽量至少提交一次，不追求数量，重在持续
- 看到对方坚持打卡时，可以顺手点个赞或留一句鼓励
- 周末可以做一次简单复盘，看看本周完成了什么、下周想补什么
